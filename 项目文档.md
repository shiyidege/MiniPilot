## 1. 整体架构与设计思路

### 思路

这是经典的**端到端自动驾驶**方案（NVIDIA PilotNet）。核心思想很简单：给神经网络看道路图像，让它直接输出方向盘角度。不需要人工设计车道检测、路径规划等中间步骤，一切由网络自己学。

### 说明

项目分四个阶段，构成完整流水线：

```
数据生成 → 模型训练 → 回归评估 → 驾驶模拟
   ↓           ↓           ↓           ↓
合成道路      PilotNet    MSE/MAE     模拟器闭环
带标注        CNN回归     分段分析    车道保持率
```

每个阶段的输入输出：

| 阶段 | 输入 | 输出 | 目标 |
|------|------|------|------|
| 数据生成 | 曲率参数 | 道路图像 + 转向角 | 创建监督学习所需的数据 |
| 模型训练 | 图像→转向角对 | 训练好的网络权重 | 最小化预测误差 |
| 回归评估 | 测试图像 | 各项误差指标 | 衡量静态预测精度 |
| 驾驶模拟 | 模型 + 测试道路 | 车辆轨迹 + 保持率 | 衡量闭环实际表现 |

项目文件结构：

```
autonomous_driving/
├── main.py                # 主入口: 调度各模块
├── config.py              # 配置: 所有可调参数集中管理
├── data_generator.py      # 合成数据: 用OpenCV画道路
├── train.py               # 训练: TensorFlow数据流水线
├── evaluate.py            # 评估: 回归指标 + 可视化

├── models/
│   └── pilotnet.py        # 模型: Keras网络定义
```

## 2. 合成数据生成器

### 思路

真实驾驶数据不好拿，但我们可以用计算机图形学画假道路。关键是：**每条假道路都知道它弯了多少**，所以能自动得到 ground truth 转向角。

道路生成的数学原理：在图像中，道路从底部延伸到地平线。每一行上道路中心的位置由曲率决定。曲率为正 → 右弯 → 转向角为正。

### 说明

`data_generator.py` 的核心是 `generate_road_image` 函数：

```python
import numpy as np
import cv2


def generate_road_image(curvature: float, road_width: float, rng) -> np.ndarray:
    """生成一张合成道路图像

    原理: 从地平线(40%高度处)到图像底部, 逐行计算道路边界。
    每行道路中心偏移 = curvature × t² × W × 0.25
      其中 t = (当前行 - 地平线行) / (总行数 - 地平线行)
      t=0 在地平线, t=1 在图像底部

    Args:
        curvature: 曲率 [-1, 1], 负=左弯, 正=右弯
        road_width: 道路宽度占图像宽度的比例 [0-1]
        rng: 随机数生成器

    Returns:
        image: (80, 160, 3), float32, [0, 1]
    """
    H, W = 80, 160
    horizon = int(H * 0.4)  # 地平线在 40% 高度处
    road_half_max = int(W * road_width / 2)

    # 创建空白图像, 填充天空和地面
    image = np.zeros((H, W, 3), dtype=np.float32)
    image[:horizon] = [0.5, 0.7, 0.9]    # 蓝色天空
    image[horizon:] = [0.3, 0.5, 0.2]    # 绿色地面

    left_bdry, right_bdry = [], []

    # 逐行计算道路边界
    for y in range(horizon, H):
        t = (y - horizon) / (H - horizon)   # 归一化位置

        # 核心公式: 道路中心偏移 = curvature × t² × 比例因子
        center_offset = curvature * (t ** 2) * (W * 0.25)
        center_x = W // 2 + int(center_offset)

        # 道路宽度: 近处宽远处窄(透视效果)
        half_w = int(road_half_max * (0.15 + 0.85 * t))

        left_bdry.append((max(0, center_x - half_w), y))
        right_bdry.append((min(W - 1, center_x + half_w), y))

    # 用多边形填充道路区域
    road_poly = np.array(left_bdry + right_bdry[::-1], dtype=np.int32)
    cv2.fillPoly(image, [road_poly], [0.31, 0.31, 0.31])  # 灰色路面

    return image
```

**关键理解**：`curvature × t²` 中 `t²` 的作用是让道路在远处开始弯曲（近处看起来更直），这模拟了真实道路的视觉透视效果。`scale = W × 0.25` 控制弯曲最大幅度。

数据增强模块用来提高泛化能力：

```python
def apply_augmentation(image, steering, rng):
    """数据增强: 模拟不同光照/天气

    修改后的转向角需要相应调整:
      如果图像向右平移了, 相当于车辆偏左,
      需要向右打方向修正 → steering += 修正量
    """
    img = image.copy()

    # 亮度调整(模拟昼夜/阴晴)
    brightness = rng.uniform(0.6, 1.4)
    img = np.clip(img * brightness, 0, 1)

    # 对比度调整
    contrast = rng.uniform(0.7, 1.3)
    mean = np.mean(img, axis=(0, 1), keepdims=True)
    img = np.clip((img - mean) * contrast + mean, 0, 1)

    # 高斯噪声(模拟摄像头噪点)
    noise = rng.normal(0, 0.02, img.shape).astype(np.float32)
    img = np.clip(img + noise, 0, 1)

    # 水平平移(模拟车辆在车道内偏移)
    shift = rng.integers(-5, 6)
    if shift > 0:
        img[:, :-shift] = img[:, shift:]
    elif shift < 0:
        img[:, -shift:] = img[:, :shift]
    # 平移时转向角要修正: 偏左(正偏移)→右打方向
    steering = np.clip(steering + shift / 160 * 0.5, -1, 1)

    return img, steering
```

## 3. NVIDIA PilotNet 模型

### 思路

NVIDIA 在 2016 年提出了这个经典架构。它的设计哲学是：**用 CNN 替代传统流水线**。传统方法需要车道线检测、路径规划、控制等多个独立模块。PilotNet 直接学"从图像到转向角"的端到端映射。

输入是前视摄像头图像，输出是 [-1, 1] 的转向角（负=左转，正=右转，0=直行）。

### 说明

`models/pilotnet.py` 中定义了两种模型。完整版 PilotNet：

```python
import tensorflow as tf
from tensorflow.keras import layers, Model


def build_pilotnet(input_shape=(80, 160, 3)) -> Model:
    """构建 NVIDIA PilotNet

    架构说明:
      5个卷积层 → 特征提取(从边缘到道路形状)
      3个全连接 → 回归(图像特征 → 转向数值)
      tanh输出  → 确保结果在[-1, 1]内
    """
    inputs = Input(shape=input_shape, name="camera_input")

    # 输入归一化(训练时统计均值和方差)
    x = layers.Normalization(name="normalize")(inputs)

    # === 卷积层: 逐层抽象 ===
    # 第一层: 检测边缘/颜色
    x = layers.Conv2D(24, (5, 5), strides=(2, 2), name="conv1")(x)
    x = layers.ELU()(x)
    # 第二层: 检测道路纹理/车道线
    x = layers.Conv2D(36, (5, 5), strides=(2, 2), name="conv2")(x)
    x = layers.ELU()(x)
    # 第三层: 检测道路边界
    x = layers.Conv2D(48, (5, 5), strides=(2, 2), name="conv3")(x)
    x = layers.ELU()(x)
    # 第四、五层: 高级特征(弯道形状等)
    x = layers.Conv2D(64, (3, 3), strides=(1, 1), name="conv4")(x)
    x = layers.ELU()(x)
    x = layers.Conv2D(64, (3, 3), strides=(1, 1), name="conv5")(x)
    x = layers.ELU()(x)

    # === 全连接层: 回归转向角 ===
    x = layers.Flatten(name="flatten")(x)
    x = layers.Dense(100, name="fc1")(x)
    x = layers.ELU()(x)
    x = layers.Dropout(0.3, name="drop1")(x)  # 防过拟合

    x = layers.Dense(50, name="fc2")(x)
    x = layers.ELU()(x)
    x = layers.Dropout(0.3, name="drop2")(x)

    x = layers.Dense(10, name="fc3")(x)
    x = layers.ELU()(x)

    # tanh确保输出在[-1, 1], 对应左满舵到右满舵
    outputs = layers.Dense(1, activation="tanh", name="steering")(x)

    model = Model(inputs=inputs, outputs=outputs, name="PilotNet")

    model.compile(
        optimizer=tf.keras.optimizers.Adam(learning_rate=1e-3),
        loss="mse",          # 回归问题用均方误差
        metrics=["mae"],     # 同时监控平均绝对误差
    )
    return model
```

另外还提供了轻量版 SmallNet（少一层卷积，全连接维度减半），在 Colab 上训练更快，精度接近。

模型编译时的关键选择：

```python
# 为什么用MSE损失?
# 转向角是连续值, 误差平方对大偏差有更高惩罚,
# 这符合驾驶场景: 小误差(差0.1)可以接受, 大误差(差0.5)可能冲出道路

# 为什么用tanh输出?
# 转向角是有范围的(-1到1), tanh天然限制在这个区间
# 如果用linear激活, 网络可能输出离谱的值(如-5或+8)
```

## 4. 训练流程

### 思路

训练流程按标准步骤：加载数据 → 构建模型 → 配置回调 → 训练 → 保存。重点在回调函数：学习率衰减（loss 不降就减半 lr）和早停（loss 连续不降就停止）。

### 说明

`train.py` 中核心是 `train_model` 函数：

```python
def train_model(model_type="smallnet", resume_path=None):
    """训练驾驶模型"""

    # 1. 加载数据(已预先生成保存为.npy)
    train_images, train_steerings = load_dataset(DATA_DIR / "train")
    val_images, val_steerings = load_dataset(DATA_DIR / "val")

    # 2. 构建模型
    model = build_model(model_type)

    # 3. 配置回调
    callbacks = [
        # 学习率衰减: val_loss 3轮不降 → lr *= 0.5
        ReduceLROnPlateau(
            monitor="val_loss", factor=0.5, patience=3, min_lr=1e-5,
        ),
        # 早停: val_loss 8轮不降 → 停止训练, 回滚到最佳权重
        EarlyStopping(
            monitor="val_loss", patience=8, restore_best_weights=True,
        ),
        # 保存最佳权重
        ModelCheckpoint(
            filepath="best.weights.h5",
            monitor="val_loss", save_best_only=True,
        ),
        # TensorBoard日志
        TensorBoard(log_dir="runs/..."),
    ]

    # 4. 创建 tf.data Dataset
    train_ds = tf.data.Dataset.from_tensor_slices((train_images, train_steerings))
    train_ds = train_ds.shuffle(30000).batch(64).prefetch(tf.data.AUTOTUNE)
    val_ds = tf.data.Dataset.from_tensor_slices((val_images, val_steerings))
    val_ds = val_ds.batch(64).prefetch(tf.data.AUTOTUNE)

    # 5. 训练
    history = model.fit(
        train_ds, validation_data=val_ds, epochs=30,
        callbacks=callbacks,
    )

    # 6. 保存模型
    model.save_weights("final.weights.h5")
    model.save("saved_model")  # SavedModel格式(可部署)
```

**训练中的自适应策略**：

```
第1-3轮:  lr=1e-3,  loss快速下降
第4-6轮:  lr=1e-3,  loss减速下降
第7轮:    val_loss 3轮不降 → lr=5e-4
第10轮:   val_loss 3轮不降 → lr=2.5e-4
...持续下降直到早停条件触发
第15轮:   val_loss 8轮不降 → 停止, 恢复最佳权重
```

## 5. 评估模块

### 思路

回归任务的评估比较直接——因为有 ground truth 转向角可以对比。评估分两个层面：

1. **静态回归指标**：看模型在测试集上的预测精度
2. **动态模拟指标**：在闭环模拟器中看实际驾驶表现

### 说明

`evaluate.py` 中的 `DrivingEvaluator`：

```python
class DrivingEvaluator:
    """驾驶模型评估器

    评估指标分三类:
      ① 回归精度: MSE/MAE/R² (直接和GT比)
      ② 分段表现: 直道vs缓弯vs急弯
      ③ 平滑度:   相邻预测的变化量
    """

    def evaluate_regression(self, images, steerings):
        """回归指标——有明确的理想值(ground truth)

        预测值 pred → 理想值 gt → 可以算误差
        """
        preds = self.model.predict(images).flatten()

        mse = np.mean((preds - steerings) ** 2)    # 均方误差
        mae = np.mean(np.abs(preds - steerings))    # 平均绝对误差

        # R² = 1 - SS_res / SS_tot
        # 衡量模型解释了多少方差, 1=完美, 0=不如直接猜均值
        ss_res = np.sum((preds - steerings) ** 2)
        ss_tot = np.sum((steerings - np.mean(steerings)) ** 2)
        r2 = 1 - ss_res / ss_tot

        return {"mse": mse, "rmse": np.sqrt(mse),
                "mae": mae, "r2_score": r2}

    def evaluate_by_curvature(self, images, steerings):
        """按弯道急缓分段评估

        考察模型在不同难度下的表现:
          - 直道:   |steering| < 0.1  → 期望误差很小
          - 缓弯:   0.1~0.4           → 可接受的误差
          - 急弯:   |steering| ≥ 0.4  → 最大的挑战
        """
        preds = self.model.predict(images).flatten()
        abs_gt = np.abs(steerings)

        results = {}
        for name, mask in [("直道", abs_gt < 0.1),
                           ("缓弯", (abs_gt >= 0.1) & (abs_gt < 0.4)),
                           ("急弯", abs_gt >= 0.4)]:
            if mask.sum() == 0:
                continue
            mse = np.mean((preds[mask] - steerings[mask]) ** 2)
            results[name] = {"count": int(mask.sum()), "mse": float(mse)}

        return results

    def evaluate_smoothness(self, images, steerings):
        """转向平滑度: 连续预测不能剧烈抖动"""
        preds = self.model.predict(images).flatten()
        diff = np.diff(preds)  # 相邻帧的预测变化量
        return {
            "jerk_std": float(np.std(diff)),        # 标准差越小越平滑
            "jerk_mean": float(np.mean(np.abs(diff))),
        }
```

评估结果的可视化：

```
预测 vs 真实散点图      误差分布直方图
  1├─ ─ ─ ─ ─ ─ ─ ─   3000├───────
   │    . . . . . . .      │  ████████
  0├───. . . . . .──   2000├──████████────
   │    . . . . . . .      │  ████████████
 -1├─ ─ ─ ─ ─ ─ ─ ─      0├──────────────
   -1    0     1          -0.5  0   0.5
   (越靠近红线越好)        (越集中越好)
```

**重要说明**：这里的 MSE、MAE 等指标是有理想值（ground truth）可减的。这和语音转换不同——语音转换没有"完美的御姐音版本"可参考。而在驾驶任务中，地面真值转向角就是正确答案，算 `(预测 - 真值)²` 是完全合理的。

## 6. 驾驶模拟器

### 思路

回归指标只能反映"单帧预测精度"——给定一张图，转向预测有多准。但在实际驾驶中，上一帧的预测结果会影响下一帧看到什么（闭环效应）。所以需要一个模拟器来测试模型的**闭环表现**。

原理是构建一条连续变化曲率的测试道路，让模型"开"过去，每步预测转向，更新车辆位置，持续追踪是否保持在道路内。

### 说明

`simulator.py` 包含三个核心类：

```python
class Track:
    """测试道路: 生成平滑变化的曲率序列"""

    def __init__(self, length=500, seed=42):
        rng = np.random.default_rng(seed)
        # 生成随机曲率后做滑动平均平滑
        raw = rng.uniform(-0.6, 0.6, length)
        self.curvatures = np.convolve(raw, np.ones(20)/20, mode="same")
        self.curvatures = np.clip(self.curvatures, -0.8, 0.8)
        self.curvatures[:10] = 0  # 开头直道
        self.curvatures[-10:] = 0  # 结尾直道

    def get_curvature_at(self, position):
        """查询指定位置的曲率"""
        return self.curvatures[min(position, len(self.curvatures)-1)]

    def render_view(self, position):
        """渲染当前位置的前视图像"""
        return generate_road_image(self.get_curvature_at(position), 0.4)
```

```python
class Simulator:
    """驾驶模拟器: 模型在环测试"""

    def run(self, model, track):
        """运行模拟

        每步执行:
          ① 渲染当前道路图像
          ② 模型预测转向角
          ③ 平滑转向(防抖动)
          ④ 更新车辆位置
          ⑤ 计算横向误差
        """
        car_x = 0.0    # 横向位置
        car_heading = 0.0  # 朝向角
        steering_smooth = 0.0  # 平滑后的转向角
        history = {"positions": [], "lateral_errors": [], ...}

        for step in range(track.length):
            # ① 获取前视图像
            image = track.render_view(step)

            # ② 模型预测
            pred = model.predict(image[np.newaxis, ...], verbose=0)[0, 0]

            # ③ 平滑: new = α × old + (1-α) × pred
            steering_smooth = 0.3 * steering_smooth + 0.7 * pred

            # ④ 更新位置: dx = sin(heading) × speed
            car_heading += steering_smooth * 0.08
            car_x += np.sin(car_heading) * 4

            # ⑤ 横向误差 = 当前位置 - 理想位置(0)
            history["lateral_errors"].append(car_x)

        return history

    def evaluate(self, history):
        """计算驾驶表现指标

        有 ground truth 道路参数可参考:
          - 应该保持在道路内(边界已知)
          - 应该在道路中心(理想位置 = 0)
        """
        errors = np.array(history["lateral_errors"])
        max_lane = 160 * 0.2 / 2  # 道路半宽(像素)

        return {
            "rms_lateral_error": np.sqrt(np.mean(errors ** 2)),
            "lane_keep_rate": np.mean(np.abs(errors) < max_lane),
            "max_deviation": np.max(np.abs(errors)),
        }
```

**模拟输出解读**：

  └── 5%的时间可能轧线或跑到路肩上

  ├── 平均偏离中心约8像素
  └── 道路半宽约16像素, 所以大部分时间在路内

**评估逻辑总结**：

| 指标 | 类别 | 理想值来源 | 能否算误差 |
|------|------|-----------|-----------|
| MSE/MAE | 回归精度 | Ground truth 转向角 | ✅ 能: `(pred - gt)²` |
| R² | 回归精度 | 完美预测=1 | ✅ 能: `1 - SS_res/SS_tot` |
| 分段MSE | 回归精度 | 各段的ground truth | ✅ 能 |
| 车道保持率 | 闭环表现 | 100% | ✅ 能: `实际 vs 100%` |
| 横向误差 | 闭环表现 | 道路中心线=0 | ✅ 能: `|position - 0|` |
| 平滑度 | 控制质量 | 无抖动=std=0 | ✅ 能: `std(diff(preds))` |

这些指标都有明确的理想值可参考，和语音转换不同。语音转换的目标状态（完美御姐音）不存在，但驾驶的目标状态（对准道路中心）是明确已知的。
